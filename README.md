дисциплины «Алгоритмы и структуры данных»

Реализовать в виде программы абстрактный тип данных «Граф» согласно варианту 
(Номер варианта – две последние цифры шифра студента (или номера зачетной книжки, студенческого). Если получившееся число превышает 50, то из него вычитается 50 – полученная разность и есть номер варианта) с учетом заданного представления графа. Операторы (операции) АТД «Граф» функционально должны выполнять следующие операции (названия операций – примерные) (1 балл из 8):
1.	FIRST(v) - возвращает индекс первой вершины, смежной с вершиной v. Если вершина v не имеет смежных вершин, то возвращается "нулевая" вершина .
2.	NEXT(v, i)- возвращает индекс вершины, смежной с вершиной v, следующий за индексом i. Если i — это индекс последней вершины, смежной с вершиной v, то возвращается .
3.	VERTEX(v, i) - возвращает вершину с индексом i из множества вершин, смежных с v.
4.	ADD_V(<имя>,<метка, mark>) - добавить УЗЕЛ 
5.	ADD_Е(v, w, c) - добавить ДУГУ (здесь c — вес, цена дуги (v,w))
6.	DEL_V(<имя>) - удалить УЗЕЛ
7.	DEL_Е(v, w) – удалить ДУГУ
8.	EDIT_V(<имя>, <новое значение метки или маркировки>) - изменить метку (маркировку) УЗЛА
EDIT_Е(v, w, <новый вес дуги>) - изменить вес ДУГИ

В рамках разработки АТД также реализовать собственный итератор для удобной работы с графом (1 балл из 8), как правило, перечисляющий узлы АТД «Граф». Использовать разработанный итератор для решения задачи, в том числе с использованием цикла range-for (for(:)– C++ или foreach()– C#) (1 балл из 8), например, в алгоритмах dfs(), bfs() (или аналогичных), которые не должны являться методами АТД «Граф» (класса «граф»), но будут использованы при решении задачи.

Реализовать задание (заданный алгоритм) (1 балл из 8) с использованием методов АТД «Граф» (1 балл из 8). Во избежание снижения баллов за работу, при реализации алгоритма решения задачи  следует обоснованно выбирать вспомогательные структуры данных (контейнеры STL, библиотечные функции и т.п.) для повышения его эффективности. Также баллы могут быть снижены, если предлагаемое решение использует представление графа отличное от заданного или обращение к данным осуществляется минуя интерфейс АТД «Граф».
